## Using Input objects \[missing assets]

Input objects define the input device and keys/buttons attached to an action. Actions are used in the State Machine \[prod doc link] to trigger an event.

<AccordionGroup>
  <Accordion title="Input object device types" defaultOpen={false}>
    Five devices are currently supported as Input objects:

    1. Nintendo Switch

    2. Xbox

    3. PlayStation 5

    4. Keyboard and mouse

    5. Quest-based controllers
  </Accordion>

  <Accordion title="Adding Input objects to a scene" defaultOpen={false}>
    Input objects can be added to a scene in two ways:

    1. Open the **Command Center**, type “input”, and select **Inputs** from the dropdown

    2. Open the **Objects** menu in the top toolbar and select **Inputs** from the dropdown
  </Accordion>
</AccordionGroup>

## Input object properties \[missing assets]

<AccordionGroup>
  <Accordion title="Attaching Input objects to actions " defaultOpen={false}>
    Open the Input section in the Properties panel

    1. Select the Input object in the Hierarchy or directly in the scene

    2. The Input section of the Properties Panel will automatically appear

    By default, there are three platforms (Desktop, Gamepad, XR) available for each Input object.&#x20;

    1. Toggle between the platforms by selecting each name in the Properties Panel

    2. Use the dropdown to define the Input object property for each action

    \[asset]
  </Accordion>

  <Accordion title="Adding and removing actions" defaultOpen={false}>
    Open the Input section in the Properties panel, then:

    * Click the **+** icon to add an action for all platforms

    * Click the **-** icon to remove a specific action from all platforms
  </Accordion>

  <Accordion title="Disabling Input properties" defaultOpen={false}>
    Open the Input section in the Properties panel and click the **power** icon to disable all Inputs.&#x20;
  </Accordion>
</AccordionGroup>
