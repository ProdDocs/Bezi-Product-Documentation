## Using Camera objects \[missing assets]

<AccordionGroup>
  <Accordion title="Adding Cameras using the Command Center" defaultOpen={false}>
    1. Select the **Bezi** icon in the top toolbar to open the **Command Center**

    2. Type "cam" into the search bar

    3. Select **Camera** from the dropdown, this will place a camera in your current view
  </Accordion>

  <Accordion title="Adding Cameras using the toolbar" defaultOpen={false}>
    1. Select the **Objects** menu from the toolbar

    2. Select **Camera** from the dropdown, this will place a camera in your current view
  </Accordion>

  <Accordion title="Renaming a Camera" defaultOpen={false}>
    1. Find the desired camera in the **Hierarchy**

    2. Double-click on the Camera name

    3. Type the new name

    4. Press `enter` to save the new name
  </Accordion>
</AccordionGroup>

## Camera view \[missing assets]

There are multiple ways to view, embody, and edit from a Camera's perspective.

<AccordionGroup>
  <Accordion title="Viewing or embodying a camera perspective" defaultOpen={false}>
    The mini viewport is located above the **Hierarchy**, in the top left corner of the Editor.&#x20;

    **To view a specific camera perspective in the viewport**: select a camera from dropdown in the bottom left corner.&#x20;

    **To embody a camera in the viewport**: select the camera icon the the bottom right corner.&#x20;
  </Accordion>

  <Accordion title="Editing the camera through the user view" defaultOpen={false}>
    * Go to the **Hierarchy** panel

    * Select the Edit camera icon&#x20;
  </Accordion>
</AccordionGroup>

## Locked and unlocked cameras \[missing assets]

Locked and unlocked cameras are key to defining a user's perspective when viewing a 3D scene. Both address specific needs based on the creators intent.&#x20;

<AccordionGroup>
  <Accordion title="Locked cameras" defaultOpen={false}>
    Locked cameras prevent a user from moving away from the camera in play mode. They are perfect for animations, game characters, and rigs.&#x20;
  </Accordion>

  <Accordion title="Unlocked cameras" defaultOpen={false}>
    Unlocked cameras allow a user to move away from the camera in play mode. They are often used as "bookmarked views" to inform users where they can go.&#x20;
  </Accordion>

  <Accordion title="How to lock or unlock a camera" defaultOpen={false}>
    1. Select the camera&#x20;

    2. Go to the right sidebar (the Properties Panel)

    3. Select the **lock** icon next to Camera
  </Accordion>
</AccordionGroup>

## Setting up cameras for a scene \[missing assets]

Every scene requires a defined camera that the user will occupy at the start, known as "start view". Once defined, users can switch between alternative cameras options in play mode.&#x20;

<AccordionGroup>
  <Accordion title="Defining the start view camera" defaultOpen={false}>
    1. Add a camera to the scene

    2. Unselect everything in the scene

    3. Go to the "Play Mode" section in the Properties Panel

    4. Click the dropdown next to "Start View" (it will say "Personal camera" by default)

    5. Select the camera you want to define as the starting view in play mode

    6. Adjust start view camera using the State Machine \[link to State Machine page]

       \[assets]
  </Accordion>

  <Accordion title="Defining other cameras that users can view in play mode" defaultOpen={false}>
    1. Add cameras to the scene

    2. Unselect everything in the scene

    3. Select **Share** in the top bar

    4. In the popover, select **Prototype settings**

    5. Confirm that "Camera bookmarks" is toggled on

    6. Select any cameras that you want to appear as camera options for users in play mode

       \[asset]
  </Accordion>
</AccordionGroup>
