## Using the Hierarchy

The Hierarchy is found in the Editor's left sidebar. Each file object is represented as a Hierarchy item.

<AccordionGroup>
  <Accordion title="Anatomy of a Hierarchy item" defaultOpen={false}>
    Each item in the Hierarchy is composed of seven elements.

    1. **Name**: the name of the object or group

    2. **Icon**: the item type (ex. Box, model, group, etc.)

    3. **Arrow**: collapse / uncollapse groups

    4. **Lightning icon**: indicates if something is interactive

    5. **Number in a circle icon**: what State an item is in (only if there's more than 1 state)

    6. **Lock toggle**: lock / unlock an item

    7. **Eye toggle**: hide / unhide an item![](/images/Hierarchy.png)
  </Accordion>
</AccordionGroup>

## Managing Hierarchy items

<AccordionGroup>
  <Accordion title="Renaming an item" defaultOpen={false}>
    There are two ways to rename an item:&#x20;

    1. Double-click on an item in the Hierarchy

    2. Double-click on the item's name on the top of the right sidebar
  </Accordion>

  <Accordion title="Lock / unlock an item" defaultOpen={false}>
    Locking an item means it will not be raycasted (unable to receive hover or select events) in the Editor.

    **To lock / unlock**: Click the **Lock** icon that appears when you hover on an item's row in the Hierarchy.&#x20;
  </Accordion>

  <Accordion title="Hide / unhide an item" defaultOpen={false}>
    Hiding an item means it will not be visible or interact-able in the Editor or in Play Mode.

    **To hide / unhide**: Click the **Eye** icon that appears when you hover on an item's row in the Hierarchy.&#x20;
  </Accordion>

  <Accordion title="Reorder an item" defaultOpen={false}>
    Click and drag items vertically to reorder them within the Hierarchy.
  </Accordion>
</AccordionGroup>
