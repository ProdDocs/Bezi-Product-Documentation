Agent Mode is Beziâ€™s _actionable_ setting. When enabled, it will make real changes in your Unity project. It does this via [scripts](https://docs.bezi.com/fundamentals/agent-mode) or Actions, which is a group of non-script based work Bezi can do for you directly in the Unity editor. 

## What Actions are available in Agent Mode today?

Bezi currently supports a growing subset of Unity editor actions. Not all Unity editor actions are supported yet, and destructive actions are intentionally limited.

<AccordionGroup>
  <Accordion title="Scene & Game Objects">
    - Create game objects
    - Update game objects
    - Delete game objects
    - Reparent game objects
    - Print game objects
    - Print game objects by path
    - Print scene hierarchy
  </Accordion>
  <Accordion title="Assets & Prefabs">
    - Create or update assets
    - Create prefabs
    - Create prefabs from assets
    - Instantiate prefabs
    - Create or update materials
    - Create or update ScriptableObjects
    - Update asset importers
    - Print assets
  </Accordion>
  <Accordion title="Components & Layout">
    - Add or update components
    - Remove components
    - Set RectTransform layout
  </Accordion>
  <Accordion title="Project & Structure">
    - Create scenes
    - Create primitives
    - Create tags
  </Accordion>
</AccordionGroup>

## How does Agent Mode's Actions work?

Part of Agent Mode's ability  Bezi can create, update, and organize scenes, game objects, prefabs, and assets on your behalf - not just explain _what_ to do, but actually do it for you in the editor
