## App Setup and Bezi \<\> Unity Connection

### What versions of Unity does Bezi support?

- **Unity 6**
- **Unity 2023**
- **Unity 2022**
- **Unity 2021**

### I see a _“Failed to install Unity Plugin”_  popup when I try connecting Bezi with my Unity project.

Occasionally this error shows if a Unity set-up isn’t conducive with the Bezi plugin. Please try these things (in order) to see if either resolve. If neither work, reach out to [support@bezi.com](mailto:support@bezi.com) for assistance.

1. Follow the Bezi flow to install the Unity plugin with administrator privileges
2. Download the Unity plugin directly from our [Changelog](https://docs.bezi.com/bezi/product/changelog)

### I can no longer find my project in Bezi after installing a package, and when I add the project manually I get a pop up that says “Error adding project”.

This error occurs when installing some template games for the Unity Asset Store. Doing so may overwrite the **productGUID** with an **invalid GUID** (00000000000000000000000000000000). Bezi requires a valid GUID to work, so this immediately breaks the connection to the project.

To fix the issue, you need to fix the project. We recommend trying these solutions, in order:

1. (If you have version control) Revert changes to ProjectSettings.asset and its .meta file
2. Close and reopen the Unity Project to regenerate the GUID
3. Delete the Library Folder to force-regenerate the GUID

<Note>
  Bezi tracks history using the GUID, so changing/regenerating the productGUID may lose your previous threads. In the future, we recommend **unchecking ProjectSettings.asset** in the Import pop-up, or copying the productGUID prior to install and replacing it after.
</Note>

### I see a _"Consent Request"_ popup from Unity

The popup is completely safe to accept and will only modify the scripts that make Bezi; none of your personal files will be affected.

We use some APIs in Bezi that can be auto-updated by Unity, but these may change between versions. This popup just updates them to use the latest for whatever version of Unity you’re working in.

### How do I know if Bezi is connected to my Unity project?

The connection indicators in your Unity project and Bezi app will both be green if they’re connected. If either are red, the connection is broken and the apps cannot communicate.

### How do I manually update or reinstall Bezi's plugin in Unity?

When a new version of Bezi is released, you'll receive an in-app prompt to update the app and plugin. Follow the instructions to do so before you start or resume working.

If Unity’s Package Manager doesn’t automatically update the plugin, follow these steps:

1. Close Bezi: right-click on the desktop app \> `Quit`
2. Open the Unity project you’ve connected with Bezi
3. Open Unity Package Manager: Window menu \> `Package Manager`
4. Delete the Bezi plugin: open “Bezi Sidekick” in the Package Manager \> `Remove`
5. Reopen the Bezi app
6. Download the most recent plugin version from our [**Changelog**](https://docs.bezi.com/bezi/product/changelog)
7. Click on the Unity project \> see the Package Manager popup
8. The Unity plugin should now be updated!

### How do I reinstate a lost connection between Unity and Bezi?

Try these common fixes. If none work, manually update the Unity plugin to restart the connection.

1. Check that you have reliable internet connection
2. Confirm both the Unity project and the Bezi app are open and connected
3. Confirm the **Project selector** dropdown shows the correct Unity project name
4. Confirm you're running a Unity version that Bezi supports
5. Close Bezi & Unity entirely
6. Restart your machine

## Bezi App

### How do I manually update or reinstall the Bezi app?

<AccordionGroup>
  <Accordion title="for macOS ">
    1. Close Bezi: right-click on the desktop app \> `Quit`
    2. Open the Applications folder \> drag Bezi.app to  Trash
    3. Restart your computer
    4. Download the latester version of the app from [https://www.bezi.com](https://www.bezi.com)

  </Accordion>

  <Accordion title="for Windows">
    1. Close Bezi: right-click on the desktop app \> `Quit`
    2. Uninstall Bezi: `Settings` \> `Apps` \> `Installed Apps` \> Remove Bezi from the list
    3. Restart your computer
    4. Download the latester version of the app from [https://www.bezi.com](https://www.bezi.com)
    5. Right-click the installer \> choose `Run as administrator`
    6. Launch Bezi
  </Accordion>
</AccordionGroup>

### I opened Bezi and got stuck on the splash screen. How do I get past it?

This can happen if the app is out of date and auto-update wasn’t able to execute. Follow these steps to manually update the app:

1. Open [**https://bezi.com/auth-app**](https://bezi.com/auth-app)
2. Select the `Launch app` button
3. Follow the prompts to update the app to the newest version

If that doesn’t work, email [**support@bezi.com**](mailto:support@bezi.com) and we can send you the latest version of the app to manually install.

## Other Common Errors, Bugs & FAQ

### My project seems to take longer to compile code and move slower after installing Bezi.

This should never happen. If you encounter it, please email [**support@bezi.com**](mailto:support@bezi.com) with a description of the problem and any screenshots of the console, profiler logs, etc..

### Why don't I see a project in the Project selector?

The **Project selector** lists all your private [Connections](/context/connections) for that Workspace. If you want a project or data source in the selector, add it as a new Connection and refresh the app.
