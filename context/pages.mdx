title: "Shared and Private Pages"
---

Pages are evergreen documentation, available to every thread in a Workspace. **Anything in a Page - coding standards, architecture decisions, design specs, system docs - is context for your Bezi agent's work anywhere in that Workspace.**

Pages can be created and modified by you or by Bezi can do it on your behalf. If you never want Bezi to create Pages, [create a Rule](https://docs.bezi.com/fundamentals/rules) explicitly stating this.

## Page documentation examples

<AccordionGroup>
  <Accordion title="Guidelines and standards ">
    When Bezi plans or implements an idea, pin @PageName so  execution follows standards.

    - UI creation standards
    - Level creation guidelines
    - Feature implementation requirements
  </Accordion>

  <Accordion title="Systems and architecture patterns ">
    Pin @PageName so Bezi  knows to follow a pre-defined pattern.

    - Core game mechanics (e.g., “Jumping uses stamina”)
    - How specific systems work 
  </Accordion>

  <Accordion title="Vision and workflow definitions">
    Store meta-level game information and pin @PageName when you need a task to keep in mind the game's end-state.

    - GDDs 
    - Instructions for Bezi to navigate your  Connections: where to look for specific information types or references
  </Accordion>
</AccordionGroup>

## Shared Pages

<Note>
  Shared Pages are only available on Team and Enterprise plans.
</Note>

Shared Pages are visible and editable by all members of a [Team Workspace](https://docs.bezi.com/fundamentals/team-workspaces). **They act as the team's single source of truth - vision, standards, rules, documented decisions - that every member's Bezi agent automatically references**. This keeps teams aligned and work consistent, even when people aren't in the same room.

If you're in a Team Workspace and need to create Pages for private use (not team-wide consumption) you can create **Private Pages**. These are visible only to the Page creator and their Bezi agent.

To update a Page from Private \<\> Shared, select the Page and drag it to the other section.

### Real-time collaboration on Shared Pages

Shared Pages support real-time collaboration. If more than one person has a Shared Page open, you will see each other’s cursors and text will appear as it’s typed.

All edits merge automatically as you work, so there's nothing to manually save and no merge conflicts to resolve.

> Bezi agents can edit Shared Pages and will not show up in real-time. All edits Bezi makes to a Shared Page will appear once it’s done editing. 

If your Team Workspace doesn't want Bezi to edit Shared Pages, each member needs to [create a Rule](https://docs.bezi.com/fundamentals/rules) explicitly stating this.

## Using Folders

Pages are organized into folders. To manage a folder or the Pages in it, hover over the Folder and click the`…`icon. You will see a menu of options to:

- Rename the folder
- Move Pages in/out of the folder
- Delete the folder

<Danger>
  Deleting a folder will delete all Pages in it. Do not delete any folders until you’ve confirmed all Pages **you want to keep** are moved.
</Danger>
